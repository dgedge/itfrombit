# Claim Status Legend

Use the same status vocabulary in script docstrings, READMEs, and paper
appendices.

| Status | Meaning |
|---|---|
| `derived` | The claim is backed by a proof, exact enumeration, or reproducible computation under stated assumptions. |
| `conditional` | The result follows if named premises or bridge assumptions hold. |
| `exploratory` | A search, probe, or model check that is useful but not canon-closing. |
| `retired` | Superseded or refuted by later audits; kept only as historical or negative-control code. |
| `imported` | Uses an established external theorem or standard numerical method rather than deriving that ingredient inside the canon. |
| `open` | A named blocker or frontier remains. |
